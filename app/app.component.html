Setup

jspm

jspm i npm:ng2-prism

npm

npm i ng2-prism --save

Usage

Features

Use a codeblock element to highlight the code within.

<codeblock language="javascript"> console.log("Hello'); </codeblock>
console.log("Hello');

Language

Use the language property to set the highlighting language. Use any of the 100+ supported languages.

<codeblock language="java"> System.out.println("java"); </codeblock>
System.out.println("java");
<codeblock language="python"> print "python" </codeblock>
print "python"
<codeblock language="go"> fmt.Println("go") </codeblock>
fmt.Println("go")

Whitespace

Whitespace is normalized, just use consistent indentation inside a codeblock.

<codeblock language="ruby"> p "Hello" p "Goodbye" </codeblock>
p "Hello" p "Goodbye"
<codeblock language="ruby"> p "Hello" p "Goodbye" </codeblock>
p "Hello" p "Goodbye"

Dynamic Loading

Use the src attribute to set a file to download as the source code for a codeblock. The language of the codeblock will be determined from the file extension, unless a language is specified.

<codeblock src="app/example/example.component.ts"> </codeblock>
<codeblock language="ruby" src="app/example/example.component.ts"> </codeblock>

Notes on Dynamic loading

  • The codeblock will automatically update on changes to src. These updates are not throttled by the codeblock.
  • The src attribute must have a file extension.
  • Everything inside the dynamic codeblock will be erased.

Line numbers

Line numbers are on by default, bind the value false to the property lineNumbers to remove them.

<codeblock [lineNumbers]="false"> No Line numbers </codeblock>
No line numbers

Themes

Use the theme property to stylize your codeblocks.

<codeblock theme="solarizedlight"> if (done) { console.log("Finished"); } </codeblock>
if (done) { console.log("Finished"); }

HTML

Use the markup language for HTML. If you use standard tags, and carefully close each one, you can write it as normal inside a codeblock.

<codeblock language="markup"> <ul class="favorites"> <li>These are</li> <li>a few of</li> <li>my favorite</li> <li>things.</li> </ul> </codeblock>
<ul class="favorites"> <li>These are</li> <li>a few of</li> <li>my favorite</li> <li>things.</li> </ul>

If you want to write a fragment of `HTML` with some unmatched tags the angular interpreter is going to fail to load your template. You must change any opening or closing tag angle brackets, <, to the html entity version &lt;.

<codeblock language="markup"> &lt;html> &lt;head> &lt;title>Title&lt;/title> ... </codeblock>
<html> <head> <title>Title</title> ...

Dynamically loaded files do not have this limitation.

Any Angular2 Components that manipulate the DOM, such as a codeblock or an ngIf, should also be escaped.

<codeblock language="markup"> &lt;div *ngIf="true" >A&lt;/div> &lt;div *ngIf="false">B&lt;/div> </codeblock>
<div *ngIf="true" >A</div> <div *ngIf="false">B</div>

Shell

Use the shell attribute to display a shell prompt. Pass in the type of shell, either bash or powershell.

<codeblock shell="bash"> ls </codeblock> <codeblock shell="powershell"> dir </codeblock>
ls dir

The language attribute is ignored on shell codeblocks.

The default theme for shells is okaidia.

Prompt

Change the prompt to whatever you want.

<codeblock shell="bash" prompt="#"> cd .. </codeblock>
cd ..
<codeblock shell="bash" prompt="[user@host] $"> cd .. </codeblock>
cd ..

Output

Shells can have certain lines treated as console output, so they don't have a prompt. Use the output attribute. It accepts a comma-separated list of lines or line ranges.

<codeblock shell="bash" output="2-5"> git status On branch master Your branch is ahead of 'origin/master' by 1 commit. (use "git push" to publish your local commits) nothing to commit, working directory clean </codeblock>
git status On branch master Your branch is ahead of 'origin/master' by 1 commit. (use "git push" to publish your local commits) nothing to commit, working directory clean
<codeblock shell="bash" output="2,4,5,7-10"> cd ../.. This is output mkdir hello so is this rm -rf hello more output more output more output more output </codeblock>
cd ../.. This is output mkdir hello so is this rm -rf hello more output more output more output more output
