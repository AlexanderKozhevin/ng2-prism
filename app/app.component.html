Setup

jspm

jspm i npm:ng2-prism

npm

npm i ng2-prism --save

Usage

Import the Codeblock.

import {Codeblock} from 'ng2-prism/codeblock';

Import the language directive for your codeblock.

import {Ruby} from 'ng2-prism/languages';

Include the Codeblock and language directive in the directives array.

@Component({ selector: 'my-component', // ... directives: [Codeblock, Ruby] })

Add a codeblock to the template and apply its language directive.

<codeblock ruby> def my_new_method p "So Impressive!" end </codeblock>
def my_new_method p "So Impressive!" end

Features

Language

Use any of the 100+ supported languages.

<codeblock java> System.out.println("java"); </codeblock>
System.out.println("java");
<codeblock python> print "python" </codeblock>
print "python"
<codeblock go> fmt.Println("go") </codeblock>
fmt.Println("go")

You may optionally specify a language attribute instead of using a directive.

<codeblock language="java"> System.out.println("java"); </codeblock>
System.out.println("java");

The attribute makes the language easy to change dynamically.

<input type="checkbox" [(ngModel)]="highlighted"> Highlighted? <codeblock [language]="highlighted ? 'typescript' : nil"> import {Component} from 'angular2/core'; </codeblock>
Highlighted?
import {Component} from 'angular2/core';

Codeblocks without a valid loaded language attribute or directive get everything except syntax highlighting.

<codeblock> Just normal text but themed with line numbers </codeblock>
Just normal text but themed with line numbers
<codeblock language="spanish"> Eso no es un lenguaje de verdad! </codeblock>
Eso no es un lenguaje de verdad!

If you choose to use the language attribute the language must still be imported, but you do not have to list it under the directives array because the template does not need to know about it.

All languages are loaded when any language is imported from ng2-prism/languages.

To import only the language you want specify the path to the language.

// if you want the directive: import {Ruby} from 'ng2-prism/languages/ruby'; // If you just want the language: import 'ng2-prism/languages/ruby';

Angular2 Binding

Use angular bindings like normal for variable output.

Name: <input type="text" [(ngModel)]="name"> <codeblock javascript> if (name === '{{'name'}}') { console.log("Hello, " + name); } </codeblock>
Name (Change Me!): if (name === '{{binding}}') { console.log("Hello, " + name); }

If you want to display the binding without processing place a pre tag around any of the braces.

<pre>{</pre>{name}} 
{
{name}}

Whitespace

Whitespace is normalized, just use consistent indentation inside a codeblock.

<codeblock ruby> p "Hello" p "Goodbye" </codeblock>
p "Hello" p "Goodbye"
<codeblock ruby> p "Hello" p "Goodbye" </codeblock>
p "Hello" p "Goodbye"

Dynamic Loading

Use the src attribute to set a file to download as the source code for a codeblock. The language of the codeblock will be determined from the file extension, unless a language is specified.

<codeblock src="app/example/example.component.ts"> </codeblock>
<codeblock ruby src="app/example/example.component.ts"> </codeblock>

Notes on Dynamic loading

  • The codeblock will automatically update on changes to src.
  • Updates to src are throttled at 300ms to prevent unnecessary http requests, you can change the time by setting debounceTime on the codeblock.
  • The src attribute must have a file extension.
  • Everything inside the codeblock will be replaced by the contents of the source file.
  • The source contents are treated as text only, not DOM elements. Components, directives, and bindings will not be processed by angular2.

Line Numbers

Line numbers are on by default, bind the value false to the property lineNumbers to remove them.

<codeblock ruby [lineNumbers]="false"> # No Line numbers </codeblock>
# No line numbers

Themes

Use the theme property to stylize your codeblocks.

<codeblock javascript theme="solarizedlight"> if (done) { console.log("Finished"); } </codeblock>
if (done) { console.log("Finished"); }

HTML

Use the markup language for HTML. If you use standard tags, and carefully close each one, you can write it as normal inside a codeblock.

<codeblock markup> <ul class="favorites"> <li>These are</li> <li>a few of</li> <li>my favorite</li> <li>things.</li> </ul> </codeblock>
<ul class="favorites"> <li>These are</li> <li>a few of</li> <li>my favorite</li> <li>things.</li> </ul>

If you want to write a fragment of HTML with some unmatched tags the angular interpreter is going to fail to load your template. You must change any opening or closing tag angle brackets, <, to the html entity version &lt;.

<codeblock markup> &lt;html> &lt;head> &lt;title>Title&lt;/title> ... </codeblock>
<html> <head> <title>Title</title> ...

Dynamically loaded files do not have this limitation.

Angular2 Components

Angular2 Components, such as a codeblock or an ngIf, will be processed by angular before highlighting.

<codeblock markup> <div *ngIf="true">A</div> </codeblock>
A

If you want to show the preprocessed version instead of the results use the escaped version.

<codeblock markup> &lt;div *ngIf="true">A&lt;/div> </codeblock>
<div *ngIf="true">A</div>

Angular2 Bindings

Pre tags are removed from markup codeblocks. If you want to show bindings without processing use the `bind` method on a local variable assigned to the codeblock.

<codeblock markup #cb>
{{bindingExpression}}
</codeblock>
{{cb2.bind('expression')}}

Shell

Use the shell attribute to display a shell prompt. Pass in the type of shell, either bash or powershell.

<codeblock shell="bash"> ls </codeblock> <codeblock shell="powershell"> dir </codeblock>
ls dir

The language attribute is ignored on shell codeblocks.

The default theme for shells is okaidia.

Prompt

Change the prompt to whatever you want.

<codeblock shell="bash" prompt="#"> cd .. </codeblock>
cd ..
<codeblock shell="bash" prompt="[user@host] $"> cd .. </codeblock>
cd ..

Output

Shells can have certain lines treated as console output, so they don't have a prompt. Use the outputLines attribute. It accepts a comma-separated list of lines or line ranges.

<codeblock shell="bash" outputLines="2-5"> git status On branch master Your branch is ahead of 'origin/master' by 1 commit. (use "git push" to publish your local commits) nothing to commit, working directory clean </codeblock>
git status On branch master Your branch is ahead of 'origin/master' by 1 commit. (use "git push" to publish your local commits) nothing to commit, working directory clean
<codeblock shell="bash" outputLines="2,4,5,7-10"> cd ../.. This is output mkdir hello so is this rm -rf hello more output more output more output more output </codeblock>
cd ../.. This is output mkdir hello so is this rm -rf hello more output more output more output more output
